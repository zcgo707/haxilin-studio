## 部署模式修正：Pages → Worker（第十轮）

### 问题
- 用户反馈一键部署后 `x.zcgo.top` 网页没有更新
- 根因：用户的 CF 架构是 **Worker 模式**（不是 Pages 模式）
  - Worker 脚本 `haxilin-studio`（7月5日创建）绑定了自定义域名 `x.zcgo.top`
  - 之前的部署用了 `wrangler pages deploy`，创建了一个**新的 Pages 项目**（7月15日），只更新了 `haxilin-studio.pages.dev`
  - Pages 项目和 Worker 是两个独立实体，Pages 部署不会更新 Worker
- 通过 CF API 排查：
  1. `GET /accounts` → 获取 account ID
  2. `GET /accounts/{id}/pages/projects` → 发现 Pages 项目只有 `haxilin-studio.pages.dev` 域名，无自定义域名
  3. `GET /accounts/{id}/workers/scripts` → 发现 Worker `haxilin-studio`（has_assets: true）
  4. `GET /accounts/{id}/workers/domains?service=haxilin-studio` → 确认 `x.zcgo.top` 绑定在 Worker 上
  5. 对比 `x.zcgo.top` 和 `haxilin-studio.pages.dev` 的 HTML 内容 → 确认版本不同

### 修复
1. **wrangler.jsonc**：`pages_build_output_dir: "./dist"` → `assets: { directory: "./dist" }`
   - 从 Pages 配置切换到 Worker 配置
2. **server.js**：部署命令 `wrangler pages deploy dist/ --project-name haxilin-studio --commit-dirty` → `wrangler deploy`
3. **package.json**：`deploy` 脚本同步改为 `wrangler deploy`
4. 清理误加的 Pages 自定义域名（通过 CF API DELETE）
5. 验证 `wrangler deploy` 成功：21 文件上传，7.94s
6. 验证 `x.zcgo.top` 返回新内容（woff2/菜单/skip-link/home-decor 全部出现）

### 关键教训
- CF 的 "Workers and Pages" 有两种部署模式，同一个 name 可以同时存在 Worker 和 Pages 项目
- Worker 自定义域名 ≠ Pages 自定义域名，两者独立
- `pages_build_output_dir` 是 Pages 专用配置；`assets.directory` 是 Worker 配置
- wrangler 检测到 `pages_build_output_dir` 会拒绝执行 `wrangler deploy`
- `wrangler deploy` 部署到 Worker，`wrangler pages deploy` 部署到 Pages 项目
